Prepare an Integration Server

  1. Assuming you are using a Windows platform, open an App Connect Enterprise Command Console.
  2. Start an integration server by entering the following command into the ACE Command Console: IntegrationServer --work-dir C:\MyServer --name MyServer --admin-rest-api 7600 --http-port-number 7800 --console-log
  3. Wait a few moments for the server to boot up on the command line, it should display: Integration server has finished initialization. The HTTP Listener has started listening on port '7600' for 'http' connections.
  4. Well done, you've just created a standalone Integration Server hosted locally on your own machine! Move onto the 'Run' tab to see a few things you can do with it.

Deploy

Follow these steps to complete the tutorial

Follow the following steps to complete the tutorial.

  1. Return your focus back to the ACE Toolkit, more specifically the Integration Explorer tab. This is located directly below the Application Development tab on the left.
  2. Right click on the 'Integration Servers' label and select 'Connect to an Integration Server'
  3. Enter the Host Name as 'localhost' and the port as '7600'. If you were to look back at the Prepare tab in this tutorial, you should notice that this is the port assigned to '--admin-rest-api' when starting the server.
  4. After clicking 'Finish' your server labelled 'MyServer' should appear with a green arrow in the thumbnail. That Green Arrow means the server is currently running, if it were a Red Cross it would signify that the server is not running. If your server doesn't appear straight away, right-click and select 'Refresh'.
  5. Now to deploy our basic application the the server using the toolkit. Locate the 'SimpleBar.bar' file in the Application Development tab, then drag and drop it onto your new server. If done correctly, the SimpleApp application should appear below the server.
  6. In order to remove resources from an Integration Server, right-click it and select 'Delete' and confirm.
